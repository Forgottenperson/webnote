git reset 使用方式

以下列程序來表示流程圖

A(Working) --[add file]--> B(Staging) --[commit]--> C(Repo)

git reset --soft (SHA1 ID)

將 HEAD 到 (SHA1 ID) 所有的 "Repo變更記錄" 恢復成 "Staging變更記錄"

git reset --mixed (SHA1 ID)

(預設) 將 HEAD 到 (SHA1 ID) 所有的 "Repo變更記錄 及 Staging變更記錄" 恢復成 "Working 變更記錄"

git reset --hard (SHA1 ID)

將 HEAD 到 (SHA1 ID) 所有的 "Repo變更記錄 及 Staging變更記錄 及 Working 變更記錄" 全部消去

git Branch

使用 branch 可以自由切換/建立新分支

但當切換的目標是 (SHA1 ID) 時

會自動產生 "未命名"(暫時性) 的 branch
